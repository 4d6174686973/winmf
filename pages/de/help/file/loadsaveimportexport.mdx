# Load/Save/Import/Export

Mit ``Load file`` werden die WinMF Dateiformate geladen. Im Zeitbereich ist die Dateiendung *.mda, im Frequenzbereich *.msp.

``Reload current file`` lädt das file mit dem Filenamen des gerade aktiven Files. In der Regel wird also das File neu geladen. Wenn mit ``QC/set current filename`` ein neuer Filename eingestellt wurde, wird das File mit diesem Namen geladen.

``Save file`` as öffnet zum Speichern einen Filebrowser. Nach dem Speichern wird der aktuelle Filename in WinMF übernommen.

``Save`` speichert unter dem gerade aktuellen Filenamen. Eine existierende Datei wird ohne Rückfrage überschrieben.  Wenn mit ``QC/set current filename`` ein neuer Filename eingestellt wurde, so wird das File unter diesem Namen geschrieben.

``Save and plot`` erzeugt zusätzlich zur Save Operation einen Plot. Eine existierende Datei wird ohne Rückfrage sowohl bei Save als auch bei Plot überschrieben (Short cut q).

``Save (autoincrement)`` speichert unter dem gerade aktuellen Filenamen. Existiert die Datei, so wird automatisch ein neues File mit diesem Filenamen und angehängter fortlaufender Nummer erzeugt. Beispiel: der aktuelle Filename ist "Test.msp". Wenn ``Save (autoincrement)`` aufgerufen wird und das file existiert in dem gerade aktuellen Ordner, dann wird ein File "Test_000001.msp" gespeichert. Beim nächsten Aufruf wird die Datei "Test_000002.msp" geschrieben, usw.

## Monkey forest

``Import *.spk`` importiert Monkey Forest Spektren.

``Export as *.spk`` exportiert Monkey Forest Spektren.

``Export *.spk`` exportiert Monkey Forest Spektren unter dem gerade aktuellen Filenamen und Pfad. Vorhandene gleichnamige Files werden ohne Rückfrage überschrieben. Sollen in Makros spk-files exportiert werden, dann muss diese Funktion dafür genutzt werden.

``Import/Export *.dat`` importiert/exportiert Monkey Forest Zeitsignale.

``Import/Export *.wav`` importiert/exportiert Wav-Dateien. Es können 16bit integer und 32bit float Waves importiert werden.

## Matlab raw data

Die Matlab Import- und Exportfunktionen erwarten im Matlab-file folgende Daten:
- data: 2-dimensionales Array, jede Zeile enthält einen Kanalvektor
- samplerate: Samplerate der Daten

Existiert die Variable 'data' nicht, dann wird der Import abgebrochen. Beim Import des Datenfeldes in den Zeitbereich wird nur der Realteil importiert.

Existiert die Variable 'samplerate' nicht, dann wird die Samplerate des gerade in WinMF angezeigten files verwendet.

``Import *.mat`` importiert Matlab Daten.

``Export as *.mat`` exportiert Matlab Daten.

``Export *.mat`` exportiert  Matlab Daten unter dem gerade aktuellen Filenamen und Pfad. Vorhandene gleichnamige Files werden ohne Rückfrage überschrieben. Sollen in Makros mat-files exportiert werden, dann muss diese Funktion dafür genutzt werden.
